## üîó Relationship Types

### One-to-Many Relationships
- **District ‚Üí Officials**: One district has many officials
- **District ‚Üí Teams**: One district has many teams
- **District ‚Üí Results**: One district can host many matches
- **Election ‚Üí Candidates**: One election has many candidates
- **Election ‚Üí Votes**: One election receives many votes

### Many-to-One Relationships
- **Official ‚Üí District**: Many officials belong to one district
- **Team ‚Üí District**: Many teams belong to one district
- **Candidate ‚Üí District**: Many candidates can represent one district
- **Vote ‚Üí Election**: Many votes belong to one election

### Self-Referencing Relationships
- **User ‚Üí District**: Users can be associated with districts
- **Result ‚Üí District**: Results can be associated with hosting districts

```sql

FOR SELECT USING (auth.uid() = id);

FOR SELECT USING (published = true);
```
